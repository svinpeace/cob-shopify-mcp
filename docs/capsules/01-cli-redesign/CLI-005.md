- **Status:** Draft

## 1. Executive Context

**Product Intent:** Schema introspection lets AI agents discover command capabilities on-demand. Instead of injecting all 49 tool schemas into the LLM context (the MCP problem), agents call `--describe` on just the command they need. This directly supports the "save context" goal of the CLI.

**System Position:** Hooks into the `--describe` global flag in `toolToCommand()` (CLI-003). Uses `zodToJsonSchema()` from CLI-001. Adds optional `outputFields` to `ToolDefinition`.

## 2. Feature Specification

**What Exists Today:**
- `tools info <name>` at `src/cli/commands/tools/info.ts` shows basic tool details (name, domain, tier, description, scopes, input keys)
- `zodToJsonSchema()` from CLI-001 can serialize Zod schemas
- `--describe` flag is defined in global flags (CLI-003) but has no handler logic yet

**What Must Be Built:**
1. `--describe` handler in `toolToCommand()`: when flag is set, serialize and output tool schema instead of executing
2. Add optional `outputFields?: string[]` to `ToolDefinition` interface at `src/core/engine/types.ts`
3. `buildDescribeOutput(tool: ToolDefinition, actionName: string, domain: string): DescribeSchema` — produces the JSON schema object
4. Add `outputFields` to existing built-in tools (at least the major ones: list_products, list_orders, list_customers, get_product, get_order, get_customer)

**User Workflow:**
```bash
$ cob-shopify products list --describe
{
  "command": "products list",
  "tool": "list_products",
  "domain": "products",
  "tier": 1,
  "description": "List products with optional filtering",
  "scopes": ["read_products"],
  "inputs": {
    "limit": {"type": "number", "min": 1, "max": 250, "default": 10, "required": false},
    "status": {"type": "enum", "values": ["ACTIVE", "DRAFT", "ARCHIVED"], "required": false}
  },
  "outputFields": ["id", "title", "handle", "status", "vendor", "productType", "totalInventory", "variants"]
}
```

**Risk Assessment:** Low. Read-only operation. Adding `outputFields` to ToolDefinition is backward-compatible (optional field).

## 1. Executive Context

**Product Intent:** Clean migration path from old CLI to new. Existing users aren't broken, but are nudged to adopt the new `<domain> <action>` pattern. Deprecated in v0.4.0, removed in v1.0.0.

**System Position:** Modifies existing CLI commands at `src/cli/commands/tools/run.ts`, `src/cli/commands/tools/list.ts`, `src/cli/commands/tools/info.ts`.

## 2. Feature Specification

**What Exists Today:**
- `tools run <name>` at `src/cli/commands/tools/run.ts` — executes a tool by name
- `tools list` at `src/cli/commands/tools/list.ts` — lists all tools
- `tools info <name>` at `src/cli/commands/tools/info.ts` — shows tool details

**What Must Be Built:**
1. Add deprecation warning to `tools run` handler: derives the new command from tool name + domain and prints to stderr
2. Add deprecation warning to `tools list` handler: suggests `cob-shopify --help` or `cob-shopify <domain>`
3. Add deprecation warning to `tools info` handler: suggests `cob-shopify <domain> <action> --describe`
4. Warning helper: `printDeprecationWarning(oldCmd: string, newCmd: string)` → `⚠ Deprecated: '<old>' → use '<new>' instead`
5. All three commands continue to execute normally after warning

**User Workflow:**
```bash
$ cob-shopify tools run list_products --limit=5
⚠ Deprecated: 'tools run list_products' → use 'cob-shopify products list' instead
# (tool executes normally, output follows)

$ cob-shopify tools list
⚠ Deprecated: 'tools list' → use 'cob-shopify --help' or 'cob-shopify <domain>'
# (list output follows)

$ cob-shopify tools info list_products
⚠ Deprecated: 'tools info list_products' → use 'cob-shopify products list --describe'
# (info output follows)
```

**Risk Assessment:** Very low. Only adds a stderr warning line. No behavior changes.

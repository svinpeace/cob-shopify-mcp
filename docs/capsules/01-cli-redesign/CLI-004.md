## 1. Executive Context

**Product Intent:** This is the capsule that makes everything visible — it wires the auto-generated commands into the CLI entry point. Users can run `cob-shopify <domain> <action>` and it just works. Domains are discovered from tool definitions. Custom YAML tools appear alongside built-ins.

**System Position:** Modifies `src/cli/index.ts` to add domain subcommands alongside existing `start`/`connect`/`config`/`tools`/`stores`. Creates `src/cli/domain-commands.ts` that groups tools by domain and produces citty parent commands.

## 2. Feature Specification

**What Exists Today:**
- CLI entry point at `src/cli/index.ts:12-27` with 5 static subcommands: `start`, `connect`, `config`, `tools`, `stores`
- All subcommands are lazy-loaded via dynamic imports
- `getAllTools()` at `src/server/get-all-tools.ts` collects all built-in tools
- `loadYamlTools()` at `src/core/registry/yaml-loader.ts` loads custom YAML tools

**What Must Be Built:**
1. `buildDomainCommands(tools: ToolDefinition[]): Record<string, CittyCommand>` — groups tools by domain, creates parent citty command per domain with action subcommands
2. Domain parent command shows help listing all actions with descriptions
3. Top-level help shows all domains with tool counts and descriptions
4. Modify `src/cli/index.ts` to dynamically add domain commands from tool registry
5. Domain descriptions map (hardcoded for built-in domains, auto-generated for custom):
   - products → "Manage products, variants, collections"
   - orders → "Manage orders, fulfillments, refunds"
   - customers → "Manage customers and segments"
   - inventory → "Track and adjust inventory"
   - analytics → "Sales reports and store analytics"

**User Workflow:**
```bash
$ cob-shopify
# Shows top-level help with domains + existing commands

$ cob-shopify products
# Shows all product actions

$ cob-shopify products list --limit 5
# Executes list_products tool

$ cob-shopify orders get --id gid://shopify/Order/123
# Executes get_order tool
```

**Risk Assessment:** Medium. This changes the CLI entry point which is the user-facing surface. Must not break `start`, `connect`, `config`, `tools`, `stores` commands.

## 1. Executive Context

**Product Intent:** The central converter that makes auto-registration possible. One function takes any `ToolDefinition` (built-in or custom YAML) and produces a complete citty command. This is the glue between the tool engine and the CLI surface.

**System Position:** `src/cli/converter/tool-to-command.ts`. Uses `deriveActionName()` and `zodToCittyArgs()` from CLI-001. Uses `OutputFormatter` from CLI-002. Calls `ToolEngine.execute()` at runtime.

## 2. Feature Specification

**What Exists Today:**
- `tools run` command at `src/cli/commands/tools/run.ts` manually wires a single tool execution. It:
  1. Loads config, creates registry, loads all tools (lines 19-49)
  2. Parses `--params` JSON and `--key=value` args (lines 71-100)
  3. Boots storage, auth, Shopify client (lines 102-118)
  4. Creates ExecutionContext and ToolEngine (lines 121-130)
  5. Calls `engine.execute()` (line 133)
  6. Outputs JSON to stdout (line 136-137)
  7. Shows cost on stderr (lines 140-145)
- This entire flow needs to be generalized into `toolToCommand()`

**What Must Be Built:**
1. `toolToCommand(tool: ToolDefinition): CittyCommandDef` — the core converter:
   - Derives action name from tool name + domain
   - Converts Zod input to citty args
   - Adds global flags: `--json`, `--fields`, `--jq`, `--describe`, `--dry-run`, `--yes`
   - Handler: boots infrastructure (lazy, shared), validates input, calls `ToolEngine.execute()`, formats output via `OutputFormatter`
2. `createExecutionContext()` — extracted from `tools/run.ts` lines 102-127, shared across all commands to avoid booting infra per-command
3. Global flag definitions as a reusable constant

**User Workflow:**
```bash
# This is what toolToCommand() enables:
cob-shopify products list --limit 5 --status ACTIVE --json
cob-shopify orders get --id gid://shopify/Order/123 --fields id,name,status
```

**Risk Assessment:** Medium. This is the critical integration point. Wrong wiring means all commands fail. Must handle infra boot errors gracefully.

- [ ] Generated command includes all global flags (--json, --fields, --jq, --describe, --dry-run, --yes)
- [ ] Running the generated command calls `ToolEngine.execute()` with validated input

- [ ] Unknown flags are rejected with helpful error message

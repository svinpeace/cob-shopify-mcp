## 1. Executive Context

**Product Intent:** Bootstrap a production-grade Node.js/TypeScript project with modern toolchain that builds, tests, lints, and is ready for npm publishing as `cob-shopify-mcp`.

**System Position:** This is the foundation layer. Every other capsule depends on this scaffold existing and being correctly configured. The toolchain choices (pnpm, tsup, Vitest, Biome, ESM-only) are locked in the architecture design doc and must not be changed.

---

## 2. Feature Specification

### What Exists Today
Nothing. The repo contains only:
- `CLAUDE.md`
- `README.md`
- `docs/` (PRDs, design doc, diagrams)

No `package.json`, no `tsconfig.json`, no source files.

### What Must Be Built
1. `package.json` — pnpm project with correct name, version, scripts, bin entry, exports, engine requirements
2. `tsconfig.json` — TypeScript 5.x, ESM-only, strict mode, path aliases (`@core/*`, `@shopify/*`)
3. `tsup.config.ts` — ESM build, entry point, .graphql raw loader support, clean output
4. `vitest.config.ts` — Path alias resolution, coverage config
5. `biome.json` — Linting + formatting rules
6. `.npmrc` — pnpm config
7. `.gitignore` — node_modules, dist, coverage, .env, .cob-shopify-mcp/
8. `.env.example` — Template with all supported env vars
9. `src/index.ts` — Minimal entry point (placeholder export)
10. Directory structure — All `src/` directories as per design doc (empty with .gitkeep)

### User Workflow
```bash
git clone <repo>
cd cob-shopify-mcp
pnpm install        # installs all deps
pnpm build          # produces dist/index.js (ESM)
pnpm test           # runs Vitest (0 tests, 0 failures)
pnpm lint           # runs Biome (0 errors)
pnpm dev            # starts dev mode with watch
```

### Risk Assessment
- **Low risk:** This is pure scaffolding, no business logic
- **Path alias risk:** `@core/*` and `@shopify/*` must resolve in both tsup (build) and Vitest (test). Misconfiguration here breaks every subsequent capsule.

## 6. Acceptance Criteria

- [ ] `pnpm install` succeeds with zero errors
- [ ] `pnpm build` produces `dist/index.mjs` (ESM format)
- [ ] `pnpm test` runs Vitest and exits with code 0 (no tests = no failures)
- [ ] `pnpm lint` runs Biome and exits with code 0
- [ ] `pnpm dev` starts tsup in watch mode without errors
- [ ] Path aliases `@core/*` and `@shopify/*` resolve correctly in both tsup and Vitest configs
- [ ] `package.json` has `"type": "module"`, `"engines": { "node": ">=22" }`, and `"bin"` entry
- [ ] TypeScript strict mode enabled
- [ ] All directories from design doc §2.1 exist under `src/`
- [ ] `.env.example` contains: `SHOPIFY_ACCESS_TOKEN`, `SHOPIFY_STORE_DOMAIN`, `SHOPIFY_API_VERSION`, `SHOPIFY_CLIENT_ID`, `SHOPIFY_CLIENT_SECRET`, `COB_SHOPIFY_READ_ONLY`, `COB_SHOPIFY_LOG_LEVEL`, `COB_SHOPIFY_STORAGE_BACKEND`
